# sksx.xlsx 相关性分析结论

基于对 109 行、39 个数值列的计算，得到 Pearson 与 Spearman 相关矩阵，并筛选出**关联较强的指标对**。完整矩阵已导出为 `correlation_pearson.csv`、`correlation_spearman.csv`。

---

## 一、关联性较强的相关指标（按业务含义分类）

### 1. 率指标之间（最有业务解释价值）


| 指标 A                       | 指标 B                       | Pearson | Spearman | 简要解读                          |
| -------------------------- | -------------------------- | ------- | -------- | ----------------------------- |
| **Challenge pass 50 rate** | **Challenge pass 75 rate** | 0.956   | 0.968    | 进度 50% 与 75% 达成率高度同步，漏斗中段一致   |
| **Challenge pass 25 rate** | **Challenge pass 50 rate** | 0.947   | 0.957    | 25% 与 50% 进度达成率强相关，前期留存好则中段也好 |
| **Challenge pass 25 rate** | **Challenge pass 75 rate** | 0.858   | 0.903    | 前期进度与后期进度强相关                  |
| **Unique redirects rate**  | **CTA click rate**         | 0.841   | 0.773    | **跳转转化率与点击率强相关**，可互为效果参考      |
| **Unique redirects rate**  | **Redirect rate**          | 0.753   | 0.963    | 唯一跳转率与总跳转率高度一致（Spearman 更强）   |
| **CTA click rate**         | **Challenge failed rate**  | 0.665   | 0.618    | 点击率高的素材，失败率也偏高（可能难度或引导不同）     |
| **Challenge solved rate**  | **Challenge pass 75 rate** | 0.650   | 0.576    | 通关率与 75% 进度率正相关               |
| **CTA click rate**         | **Redirect rate**          | 0.532   | 0.725    | 点击率与跳转率中等相关                   |


**结论**：  

- **跳转转化率 (Unique redirects rate)** 与 **CTA 点击率 (CTA click rate)** 关联最强（约 0.84），可视为同一转化维度的不同度量，做归因时注意不要重复加权。  
- **游戏进度各阶段达成率**（25% / 50% / 75%）之间高度相关，说明「要么整体留存好，要么整体流失多」，可考虑用 1～2 个代表指标做监控。

---

### 2. 结构性/漏斗内强相关（计数类，多为同一漏斗上下游）

以下多为「同一漏斗、不同阶段」的计数，数值规模高度同步，Pearson 接近 1 属正常：

- **HTML loading / HTML loaded / HTML displayed**：三者几乎完全共变（1.00），可视为同一加载漏斗。
- **Challenge started** 与 **Challenge pass 25/50/75**、**CTA clicked**、**HTML loaded/displayed**：强正相关（0.9+），体现「曝光大 → 各环节绝对量都大」。
- **Impressions** 与 **Spend / Unique redirects / Redirect count / Unique interactions / Total interactions**：接近 1，说明花费与互动量主要由展示量驱动。
- **Unique redirects** 与 **Redirect count**：0.985（Spearman），唯一跳转与总跳转高度一致。
- **Challenge started** 与 **Challenge failed / Challenge solved**：0.95+，开始人数与成败人数强相关。

**结论**：计数类指标之间高相关主要反映「规模效应」和「漏斗结构」，做建模时注意多重共线性，可只保留少数代表列（如 Impressions、CTA clicked）或改用率指标。

---

### 3. 错误类与规模

- **Unique interactions / Total interactions / Redirect count** 与 **Black view error、Runtime error**：Pearson 约 0.99，说明错误数随互动量、跳转量线性增长，可能是「量越大报错次数越多」而非单次出错概率变化。
- **Rendering error** 与上述规模类指标也高度相关，但 **Spearman 仅 0.13～0.22**，说明排序关系弱，渲染错误更可能和素材/环境有关，而非单纯规模。

## 二、建议用法

1. **做归因或效果评估**：优先用 **Unique redirects rate** 或 **CTA click rate** 之一作为「转化率」主指标，避免两个同时进模型导致共线性。
2. **做游戏难度/留存分析**：用 **Challenge pass 25/50/75 rate** 中 1～2 个即可，三者高度相关。
3. **做异常与质量监控**：错误数建议看**错误率**（如 Runtime error rate）与展示/互动的关系，而不是只看绝对错误数（易被规模主导）。
4. **建模/回归**：计数类指标间多重共线性明显，可只保留 Impressions（或 Spend）、CTA clicked 等少数计数，其余用率指标或做标准化/降维。

- 运行脚本：`python correlation_analysis.py`
