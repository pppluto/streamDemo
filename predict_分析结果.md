# 预测分析结果

基于 **tag.xlsx Sheet2** 的入门级预测分析（5 折交叉验证）。

## 一、设定

- **特征**：点消、拖消、目标物品、完播率、挑战通过率(25/50/75)、平均时长、各类报错率。
- **回归目标**：`Unique redirects rate`（跳转转化率 CVR）。
- **分类目标**：CVR 是否高于中位数（高/低转化）。

## 二、结果摘要

### 回归（预测 CVR）

- 有效样本数：33
- R²：-1.0233（越接近 1 拟合越好，样本少时易偏低）
- MAE：0.089469
- RMSE：0.119709

### 二分类（高/低 CVR）

- 有效样本数：37
- Accuracy：0.8107
- F1 (weighted)：0.8106
- ROC AUC：0.8483

## 三、指标含义说明

### 回归用到的指标


| 指标       | 含义                                                 | 你这边的结果怎么看                                                 |
| -------- | -------------------------------------------------- | --------------------------------------------------------- |
| **R²**   | 模型解释了多少「目标变量的波动」。1 = 全解释，0 = 和瞎猜均值差不多，负值 = 比猜均值还差。 | -1.02：在当前样本和特征下，**无法可靠地预测 CVR 的具体数值**，属正常（样本少、特征共线性强时常见）。 |
| **MAE**  | 预测值相对真实值，平均差多少（绝对值）。单位与 CVR 一致（你这边 CVR 是 0～1 的小数）。 | 约 0.089：平均误差约 **8.9 个百分点**（若 CVR 用百分比表示则约 8.9%）。          |
| **RMSE** | 类似 MAE，但对大误差更敏感。                                   | 约 0.12：大误差时偏差约 **12 个百分点**。                               |


结论：**回归**更适合将来样本更多、特征更稳时再用来预测「具体 CVR 数值」；当前更有用的是下面的**分类**结果。

### 二分类用到的指标


| 指标                | 含义                                         | 你这边的结果怎么看                                         |
| ----------------- | ------------------------------------------ | ------------------------------------------------- |
| **Accuracy**      | 预测「高 CVR / 低 CVR」时，猜对的比例。                  | 约 **81%**：10 条素材里大约 8 条能正确判断是高转化还是低转化。            |
| **F1 (weighted)** | 综合「有没有漏掉高转化」「有没有误判成高转化」的平衡分数，1 最好。         | 约 **0.81**：和准确率接近，说明高/低两类都判得比较均衡。                 |
| **ROC AUC**       | 模型区分「高 CVR vs 低 CVR」的能力，0.5 等于随机，1 等于完全区分。 | 约 **0.85**：**区分能力不错**，能较好地区分「更容易高转化」和「更容易低转化」的素材。 |


结论：用「标签 + 完播率 + 挑战通过率 + 报错率」等，**判断一条素材更偏高 CVR 还是低 CVR** 是可行的，对后续素材方向有参考价值。

---

## 四、对后续素材方向的指导意义

1. **高/低转化是可区分的**
  分类 AUC≈0.85 说明：现有特征（点消/拖消/目标物品、完播率、挑战通过率、报错率等）里，已经包含与「高 CVR vs 低 CVR」相关的信息。做新素材或选品时，可以优先考虑历史上更常出现在「高 CVR」一侧的配置（例如某些标签、完播与通关表现更好的类型）。
2. **不必强求预测具体 CVR 数值**
  当前回归 R² 为负，说明在现有数据规模下，精确预测「CVR 会是多少」还不现实。更适合把分析用在：**判断素材更可能落在「高转化组」还是「低转化组」**，用于排序、筛选或分层投放，而不是追求具体数字。
3. **和既有标签结论对齐使用**
  你之前的结论里：**目标物品** 展示/花费/点击量明显更高、通关率更好；**点消** CTR/跳转率更高但通关率偏低；**拖消** 完播率明显偏低。  
   预测分析补充的是：这些特征组合在一起，能较好地区分「高 CVR / 低 CVR」。后续可以：  
  - 对**目标物品**类素材适当倾斜资源（与「高消耗、高转化」一致）；  
  - 对**点消**优化游戏难度/引导，看能否拉高通关率同时保持 CTR；  
  - 对**拖消**重点排查完播与体验，再观察 CVR 是否改善。
4. **后续可做的动作**
  - 新素材上线前：用现有模型打一个「高/低 CVR」倾向分，作为冷启动阶段的参考（可随数据增多定期重训）。  
  - 积累更多素材与数据后：再尝试用回归预测具体 CVR、或换更复杂模型，并做特征重要性分析，看哪些维度对转化影响最大。

## 五、如何继续做预测分析

1. **换预测目标**：在脚本中修改 `TARGET_REGRESSION` / `TARGET_BINARY`，例如改为 `CTA click rate`、`HTML completion rate`、`Challenge pass 50 rate` 等。
2. **增加/删减特征**：修改 `FEATURE_COLS`，注意避免「用目标或强相关结果当特征」（信息泄漏）。
3. **换模型**：可尝试 `RandomForestRegressor` / `RandomForestClassifier`（样本少时易过拟合，建议用 CV 看泛化）。
4. **样本增多后**：再做特征筛选（如基于相关性或模型 feature_importance）、调参，或划分固定测试集评估。
